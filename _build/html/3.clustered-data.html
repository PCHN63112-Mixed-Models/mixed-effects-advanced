Building the Two-level Clustered Model#

We will use the steps described in the previous part of the lesson to build this model. We will detail each one so you get a good idea of how the process applies to this dataset. The aim is to build intuition and confidence so that you can do the same for any dataset.

1. Identify the Structure#

We already discussed identifying the structure of a dataset at the beginning of this lesson. We are also organising these examples by structure, so this first step is somewhat redundant. However, we will keep it here for completeness.

To begin, we need to identify our unit of analysis. This is the element of the data that we are trying to make inferences on. So, in this example, our units are individual rat pups. Our primary question is about how the treatments affect the weight of the rat pups.

In the second step, we need to determine what each row of the data represents. As we can see above, each row corresponds to only a single unit measured once. So this is not repeated measures or longitudinal data, but it could be clustered data.

In the final step, we need to determine if there are any clusters. If there are, we use a multilevel/LME approach. If there are not, the data are cross-sectional and we go back to normal linear regression. So, we look and see whether there are any variables that are constant across multiple units. In this example, there are several: sex, litter, litsize and treatment. The important question is whether any of these represent a shared latent influence that could affect the correlation between pups? If we think back to our sticky notes vs rubber bands metaphor, what elements of the data tie all those shared units together and which are simply labels that we give those units?

In this example, sex, litsize and treatment are all sticky notes. They are labels given to each individual unit. They describe a characteristic of the pup or a characteristic of their environment, but they do not tie them together. By comparison, litter itself is a rubber band. All pups from the same litter are tied together by the biological reality of sharing a mother, genes, siblings and rearing environment. So, our conclusion is that litter is a clustering variable. All pups from the same litter are therefore expected to be correlated to some degree.

More on Litter Size

You may find litsize a confusing inclusion because it refers to litter and thus sounds like it should relate to clustering. However, this is a description rather than a structure. The way to think about it is if, hypothetically, litter size was a rubber band it would mean that all pups from litters of the same size were connected, irrespective of their mother, rearing environment or siblings. So, it is simply the total number of pups in the litter that matters biologically, rather than anything else. Thus all pups from a litter of size 8 would be connected together. Their mother, genetics and other factors do not matter. Consider for yourself whether this sounds plausible and thus whether you think litsize should be treated as a rubber band or a sticky note.

2. Model a Single Dependency Structure#

In our second step, we start by building a model for a single dependency structure. With clustered data, the entities that create dependencies are the clusters. As such, a single dependency structure can be taken to mean a single cluster. So, we start by writing the model for one cluster. In doing so, we start by imagining that our data was only collected from a single litter of pups.

We can look at this in R by subsetting the data to just those values from the first litter

litter.1 <- subset(ratpup, litter=='1')

As we can see, litter, litsize and treatment are all constant at the level of a single litter and so cannot be used as predictors. This make sense because these either index the data as a whole (litter) or are manipulated across litters (litsize,treatment). So, if we only have one litter, these are irrelevant. We could even go as far as removing these from litter.1. If you are working with very complex datasets, this is something you may want to do to help clarify the model you are building. However, for this example, we will leave the data as-is.

Based on the above, the only useable variables we have at the level of a single litter are weight and sex. Because there are no repeated measurements, pup_id simply indexes the rows of the data frame and is not of much use. So, based on all this, the only model we can build at the level of a single litter is one that models weights as a function of the sex of the pup. This is not wholly unreasonable, as we expect male pups to be heavier than female pups. So our starting point is the normal linear model

\[ y_{ij} = \mu + \alpha_{j} + \eta_{ij}, \]

where \(y_{ij}\) is the value of weight for pup \(i\) of sex \(j\), \(\mu\) is the average weight of all pups in this litter, \(\alpha_{j}\) is the effect of sex at level \(j\) and \(\eta_{ij}\) is the error. This is effectively a one-way ANOVA model of sex within a single litter.

We can run this model in R to check that we have a model that is possible with a single litter. This is not wholly necessary, but can be a useful exercise when building these models up slowly. It is also useful because we can see the model formula for one litter, which will become relevant a little later on.

lm.litter.1 <- lm(weight ~ 1 + sex, data=litter.1)

3. Add Multiple Dependency Structures#

For step 3, we need to imagine that we had multiple litters and then need to rewrite the model above accordingly. To begin, we need another index to make it clear which litter the model is referring to. To do this, we will use \(k\) to indicate litter, but will use a slightly different notation to illustrate which elements belong to the model for litter \(k\). So, we write the model for litter \(k\) as

\[ y^{(k)}_{ij} = \mu^{(k)} + \alpha^{(k)}_{j} + \eta^{(k)}_{ij} \]

This indicates that everything in the model depends upon the specific litter. This is generally our theoretical starting-point. However, in the next step, we will determine whether everything here can change within each litter, given the data we have, and whether we want them to change with each litter. We have already gone some way towards this in the previous step by fitting the model for a single litter, but we will formalise it now.

4. Identify Terms that Change with the Dependency Structure#

Now that we have our model for each level of litter, we need to make some decisions. As indicated above, every element has the potential to be different for each litter. Of course, the data itself always will be, as will the errors. So our question is about how to treat those elements that form the mean function of each litter:

\[ E\left(y^{(k)}_{ij}\right) = \mu^{(k)} + \alpha^{(k)}_{j} \]

For each of these term, we have a choice. Do we imagine that this effect changes with each litter, or do we want to treat it as a global effect that is assumed to be the same across all litters? In other words, we are trying to capture why the litter variable matters. What influence does litter have on the weight of the pups within it? Any element that changes from litter-to-litter captures why litter influences weight, whereas any element that is the same suggests that litter is irrelevant.

As will hopefully become clear, there is no absolute truth here. These decisions have to be made by reasoning about the problem and considering what the model is saying, given the choices we make. Although we can use model comparisons to see which choice is most compatible with the data, we also need to start from a position of justifying our choices. So there is no shortcut to engaging your brain and thinking about the problem in front of you. This can be difficult, but is part of the reality of analysing data and part of your worth as a data analyst.

How to Treat \(\mu^{(k)}\)?#

To start with, let us consider how to treat \(\mu^{(k)}\). This is the average weight per-litter. So, from a purely theoretical perspective, do we think that each litter should get its own baseline weight, or do we want to constrain the model so that all litters share the same baseline weight? In other words, do we think that the reason that litter is important is because it influences the average weight of the pups within each litter, or do we think that each litter has the same baseline weight and any deviations are sampling noise that is unrelated to litter? For this model, we choose to allow the baseline weight to vary and so our model per-litter remains

\[ E\left(y^{(k)}_{ij}\right) = \mu^{(k)} + \alpha^{(k)}_{j} \]

The reasoning behind this is explored in the drop-down below

Reasoning About the Intercept

Our choice in terms of the intercept \(\mu^{(k)}\) is effectively a decision between the following models. Either we allow \(\mu\) to change from litter-to-litter

\[ E\left(y^{(k)}_{ij}\right) = \mu^{(k)} + \alpha^{(k)}_{j} \]

or we fix \(\mu\) to the same value for each litter

\[ E\left(y^{(k)}_{ij}\right) = \mu + \alpha^{(k)}_{j}. \]

To think about this more clearly, imagine that we were able to take two mother rats and make them give birth to new litters over and over again. Our key question is whether we believe that

  1. Over time the average weight of all the litters, irrespective of their mother and rearing environment, will converge on the same value. In other words, the mother and rearing environment does not matter. Any discrepancy in our measurements is simply sampling variation that is unrelated to each mother and rearing environment.

  2. Over time, the average weight of all litters from the same mother will converge on their own unique value that will be different from another mother. In other words, the mother and rearing environment does matter. For instance, mother rat A always produces litters that are heavier on average than mother rat B. Any discrepancy in our measurements is then a mixture of sampling variation and the effect of a specific mother and rearing environment.

This is precisely the choice between treating the mean weight per-litter as a constant across litters or as unique per-litter. Our question is which of these descriptions of reality is more plausible?

In this example, it is probably not sensible to constrain all litters to the same expected weight. Depending upon different genetic factors and birthing conditions, some litters will be heavier or lighter than others. This goes beyond just random sampling noise and becomes a systematic source of variation that directly relates to the different litters. In fact, this was the whole reason for characterising litter as a clustering variable in the first place. So, in this instance, we will argue that it makes most sense to allow \(\mu^{(k)}\) to differ per-litter. Irrespective of whether we are dealing with clusters or subjects, it is almost always sensible to start with this random-intercepts model.

Once we have made this decision, we also need to examine the data from a practical perspective, to see if it can support a different average value of weight per-litter. Even if we want to do this, that data needs to allow it. This requires at least two values of weight for each litter, so that there is variance left over once \(\mu^{(k)}\) has been fit. In this example, each litter has multiple values and so a varying-intercept is perfectly possible. So, we conclude that both theoretically and practically we can allow a different baseline weight per-litter.

How to Treat \(\alpha^{(k)}_{j}\)?#

Now, we need to provide the same consideration to \(\alpha^{(k)}_{j}\). Remember, this is the effect of sex on weight per-litter. Unlike the random-intercepts, the decision about how to treat \(\alpha^{(k)}_{j}\) is less clear-cut. If our bare minimum is include an intercept per-litter, then we can still capture the importance of litter whether we also allow \(\alpha^{(k)}_{j}\) to vary or not. So this choice is much more about our reasoning than it is about the need to capture litter as a clustering variable. For the purpose of this analysis, we will start by assuming that the effect of sex can differ per-litter and so our model remains

\[ E\left(y^{(k)}_{ij}\right) = \mu^{(k)} + \alpha^{(k)}_{j} \]

This is again explored more in the drop-down below

Reasoning About the Slope

As with the intercept parameter, we will consider our two modelling options. If we allow the effect of sex to differ per-litter, we are assuming

\[ y^{(k)}_{ij} = \mu^{(k)} + \alpha^{(k)}_{j} + \epsilon^{(k)}_{ij} \]

Alternatively, if we fix the effect of sex to be the same across all litters, we are assuming

\[ y^{(k)}_{ij} = \mu^{(k)} + \alpha_{j} + \epsilon^{(k)}_{ij}. \]

So, our question is one of reasoning. Do we think that there is a universal effect of sex on weight that is always there irrespective of genetics or rearing environment? Or, do we believe that the magnitude of the effect of sex on weight depends, at least to some degree, on both genetics and rearing environment?

Of course, the problem here is that you could argue it either way. It very much depends on how strongly we believe that the effect of sex on weight is homogeneous across litters, or how strongly we believe that there could be a gene \(\times\) environment interaction that change the effect of sex for each litter? Is it plausible that some mother rats may consistently have female pups that are heavier than male pups? Or female and male pups that are largely equivalent in weight? Or, is it more plausible to think that male pups will always be heavier than female pups on average, irrespective of mother or rearing environment?

When faced with this sort of decision, we either have to (a) pick one possibility that we believe more strongly or (b) pick the most general case and then use model comparisons to see which the data supports. If you are working alongside domain experts on the analysis, this is the sort of conversation you could have with them to pick the option that is most plausible. In the absence of this, we will take the second approach, which is to temporarily assume the more general case of the sex effect differing per-litter, but with a view to using model comparisons for the final decision on this[1].

Once we have made this decision, we need to examine the data from a practical perspective, to see if it can support a different sex effect per-litter. In order to do so, we need at least two male pups and two female pups on average per-litter. If each litter only consisted of a single male and a single female, this would not be possible. Similarly, if each litter consisted of only a single sex, this would also not be possible. We can have a look at the average number of male and female pups across litters using

colMeans(xtabs(~ litter + sex, data=ratpup))

  Female     Male 
5.592593 6.333333

This indicates that there should be enough variation on average per-litter to allow this. Even if there was not, we would soon find our because the model would fail to fit. However, we can at least avoid this embarrassment by trying to reason about it first. Although tempting to brute-force these decisions by examining when the model fails, this is not a good strategy because it suggests that you do not understand the model, nor why it is failing to fit. You should have a clear idea abut this before even approaching the software.

5. Write the Higher-level Models#

In the previous step, we determined whether to allow certain terms to vary per-litter, or whether we want to treat them as global effects irrespective of litter. What we were actually doing here was deciding whether a term was to be treated as random or fixed. Because we decided to treat both \(\mu^{(k)}\) and \(\alpha_{j}^{(k)}\) as varying per-litter, then these are both random variables. For every new litter we sample, their values will change. This means that these terms must have associated probability distributions that describe their behaviour over repeated sampling. For simplicity, if we assume these distributions are normal, our model is therefore

\[\begin{split} \begin{alignat*}{1} y^{(k)}_{ij} &= \mu^{(k)} + \alpha^{(k)}_{j} + \eta^{(k)}_{ij} \\ \mu^{(k)} &\sim \mathcal{N}(\mu, \sigma^{2}_{2}) \\ \alpha^{(k)}_{j} &\sim \mathcal{N}(\alpha_{j}, \sigma^{2}_{3}) \end{alignat*} \end{split}\]

So, the average weight of each litter is conceptualised as being drawn from a larger population of average weights, with some population mean weight of \(\mu\). Similarly, the effect of sex within each litter is conceptualised as being drawn from a larger population of sex effects, with some population mean effect of \(\alpha_{j}\).

Remembering that we can always write probability models in terms of expressions for the means plus random error, we can also rewrite the above as

\[\begin{split} \begin{alignat*}{1} y^{(k)}_{ij} &= \mu^{(k)} + \alpha^{(k)}_{j} + \eta^{(k)}_{ij} \\ \mu^{(k)} &= \mu + \xi^{(k)} \\ \alpha^{(k)}_{j} &= \alpha_{j} + \phi^{(k)}_{j} \end{alignat*} \end{split}\]

So, now we have two extra error terms in the form of \(\xi^{(k)}\) and \(\phi^{(k)}_{j}\). The assumptions about all the error terms are then

\[\begin{split} \begin{alignat*}{1} \eta^{(k)}_{ij} &\sim \mathcal{N}(0,\sigma^{2}_{\eta}) \\ \xi^{(k)} &\sim \mathcal{N}(0,\sigma^{2}_{\xi}) \\ \phi^{(k)}_{j} &\sim \mathcal{N}(0,\sigma^{2}_{\phi}) \end{alignat*} \end{split}\]

So we can see that what we have ended up with is a model containing three variances. We have partitioned the overall variance of the data into 3 components

\[ \text{Var}\left(y\right) = \sigma^{2} = \sigma^{2}_{\eta} + \sigma^{2}_{\xi} + \sigma^{2}_{\phi}. \]

Notice that we did not start by doing this. This is simply where we have ended-up by building the model slowly in a principled fashion. This structure has simply fallen-out of the decisions we have made along the way. So, our current conceptualisation is of a two-level model of the following form

\[\begin{split} \begin{alignat*}{2} \text{Level 1:} \\ y^{(k)}_{ij} &= \mu^{(k)} + \alpha^{(k)}_{j} + \eta^{(k)}_{ij} &\quad\small{\text{Weight of pup $i$ of sex $j$ in litter $k$}} \\ \quad \\ \text{Level 2:} \\ \mu^{(k)} &= \mu + \xi^{(k)} &\quad\small{\text{Average weight of litter $k$}} \\ \alpha^{(k)}_{j} &= \alpha_{j} + \phi^{(k)}_{j} &\quad\small{\text{Effect of sex $j$ in litter $k$}} \end{alignat*} \end{split}\]

However, there is more we can now do with the two 2nd-level models.

Modelling \(\mu^{(k)}\)#

Let us start by focussing on the 2nd-level model of \(\mu^{(k)}\). If we treat it like any other regression model, we can decide whether there is more we can include to model the mean weight of litter, beyond just the population mean \(\mu\). So, let us pretend we only have the model

\[ \mu^{(k)} = \mu + \xi^{(k)}. \]

Is there anything else in this dataset that we think might relate to the mean weight of each litter? Well, remember in this experiment that the experimental treatment was applied to litters individually. So, an additional element that might affect the average weight in litter \(k\) is the treatment they were given. If we denote the effect of the treatment that has been applied \(\gamma_{l}\), then our model of the mean becomes

\[ \mu^{(k)} = \mu + \gamma_{l} + \xi^{(k)}. \]

If we imagine, for simplicity, that there are 4 litters where the first two are given treatment 1 and the last two are given treatment 2, this model implies

\[\begin{split} \begin{alignat*}{1} \mu^{(1)} &= \mu + \gamma_{1} + \xi^{(1)} \\ \mu^{(2)} &= \mu + \gamma_{1} + \xi^{(2)} \\ \mu^{(3)} &= \mu + \gamma_{2} + \xi^{(3)} \\ \mu^{(4)} &= \mu + \gamma_{2} + \xi^{(4)} \\ \end{alignat*} \end{split}\]

Sometimes, the clearest picture can come from simply expanding the indices and seeing what the model is implying.

Modelling \(\alpha^{(k)}_{j}\)#

Now, let us move on to focussing on the 2nd-level model of \(\alpha^{(k)}_{j}\). Again, if we treat this like any other regression model, we can decide whether there is more we can include to model the effect of sex within each litter, beyond the population effect \(\alpha_{j}\). So, let us pretend we only have the model

\[ \alpha^{(k)}_{j} = \alpha_{j} + \phi^{(k)}_{j} \]

Is there anything else in this dataset that we think might relate to the effect of sex on the average weight in this litter? Well, again, we might want to consider the treatment was applied. If the treatment works differentially in male vs female pups then there could be a treatment \(\times\) sex interaction effect. If this is of interest, we can expand this model to

\[ \alpha^{(k)}_{j} = \alpha_{j} + (\alpha\gamma)_{jl} + \phi^{(k)}_{j} \]

So this is saying that the unique effect of sex in litter \(k\) is governed by the effect of sex (\(\alpha_{j}\)), plus the interaction between treatment and sex (\((\alpha\gamma)_{jl}\)). As with before, it can be helpful to expand the indices here. For each litter \(k\) we have an additional index for sex in the form of \(j\). So, for 4 litters given 2 different treatments, we have

\[\begin{split} \begin{alignat*}{1} \text{Litter 1 Treatment 1} \begin{cases} \alpha_{1}^{(1)} &= \alpha_{1} + (\alpha\gamma)_{11} + \phi^{(1)}_{1} \quad\text{Male pups} \\ \alpha_{2}^{(1)} &= \alpha_{2} + (\alpha\gamma)_{21} + \phi^{(1)}_{2} \quad\text{Female pups}\\ \end{cases} \\ \text{Litter 2 Treatment 1} \begin{cases} \alpha_{1}^{(2)} &= \alpha_{1} + (\alpha\gamma)_{11} + \phi^{(2)}_{1} \quad\text{Male pups} \\ \alpha_{2}^{(2)} &= \alpha_{2} + (\alpha\gamma)_{21} + \phi^{(2)}_{2} \quad\text{Female pups} \\ \end{cases} \\ \text{Litter 3 Treatment 2} \begin{cases} \alpha_{1}^{(3)} &= \alpha_{1} + (\alpha\gamma)_{12} + \phi^{(3)}_{1} \quad\text{Male pups} \\ \alpha_{2}^{(3)} &= \alpha_{2} + (\alpha\gamma)_{22} + \phi^{(3)}_{2} \quad\text{Female pups} \\ \end{cases} \\ \text{Litter 4 Treatment 2} \begin{cases} \alpha_{1}^{(4)} &= \alpha_{1} + (\alpha\gamma)_{12} + \phi^{(4)}_{1} \quad\text{Male pups} \\ \alpha_{2}^{(4)} &= \alpha_{2} + (\alpha\gamma)_{22} + \phi^{(4)}_{2} \quad\text{Female pups} \\ \end{cases} \\ \end{alignat*} \end{split}\]

This looks complicated, but it is the simple expansion of the model above, where we replace the indices with actual numbers. You can spend some time studying this, but do not get too bogged down in it.

6. Collapse to Form the Mixed-effects Model#

… So, at present, our two-level model is

\[\begin{split} \begin{alignat*}{1} \text{Level 1:} \\ y^{(k)}_{ij} &= \mu^{(k)} + \alpha^{(k)}_{j} + \eta^{(k)}_{ij} \\ \quad \\ \text{Level 2:} \\ \mu^{(k)} &= \mu + \gamma_{l} + \xi^{(k)} \\ \alpha^{(k)}_{j} &= \alpha_{j} + (\alpha\gamma)_{jl} + \phi^{(k)}_{j} \end{alignat*} \end{split}\]

…

\[ y^{(k)}_{ij} = \mu + \alpha_{j} + \gamma_{l} + (\alpha\gamma)_{jl} + \xi^{(k)} + \phi^{(k)}_{j} + \eta^{(k)}_{ij} \]

…

\[ y_{ijk} = \mu + \alpha_{j} + \gamma_{l} + (\alpha\gamma)_{jl} + \xi_{k} + \phi_{jk} + \eta_{ijk} \]

Where we end up with what is essentially a two-way ANOVA model with three error terms.

We can now collapse these levels together to give the mixed-effects specification.
